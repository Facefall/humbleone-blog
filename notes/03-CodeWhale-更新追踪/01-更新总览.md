## 对使用者/维护者的判断

### 风险

- 如果只盯版本号，会漏掉这种“同一版本说明被继续修订”的更新
- 如果把 `0.8.61` 当成普通补丁版，可能会低估其对多 Agent 运行时语义的影响
- 由于官方 changelog 会在既有区块中增补细节，团队内部若做升级评估，最好保留每日快照而不是只保留最新链接

### 机会

- 这类持续修订通常说明维护者仍在澄清能力边界、补齐归因或完善运行时叙述
- 对依赖 sub-agent、goal mode、provider routing 的使用者，这比单纯“新版本发布”更适合作为回归测试信号
- 当前 `Unreleased` 尚未进入新版本号，意味着后续还有可能在同一主题上继续扩充

## 建议动作

1. 继续把 changelog 哈希变化视为一级信号，而不是只看版本号。
2. 如果你在评估 CodeWhale 的多 Agent 能力，优先复查 `0.8.61` 涉及的 worker lifecycle、goal mode、route isolation 假设。
3. 如果后续要做更精确的“改了哪几句”追踪，自动化私有 state 需要额外保存上一份完整 changelog 快照或结构化分节摘要。
